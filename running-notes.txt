Storage classes
===============
AWS charge us for
1. storing the data
2. retrieve the data

Deciding point is how frequently you access data?

from top to down, retrievals become slower and storage become cheaper

Standard -> retrievals are free, but cost for storage
Standard-IA -> not frequent access, once in a month.. storage becomes cheaper than standard. retrievals become costly than standard
One zone standard-IA -> 1AZ. danger. build files, cache files, etc.. if you are able to regenerate
Glacier Instant -> storage becomes cheaper, retrievals becomes instant but costly -> seconds
Glacier Flexible Retrieval -> storage becomes more cheaper, retrievals becomes more slow but more costly -> minutes
Glacier Deep archieve -> storage becomes cheapest, but retrievals are costlier and slow usually in hours

app logs
========
running logs -> standard. if any problem we need immidiate retrievals and frequent retrievals
1 month old logs -> standard-IA. because storage is chaper, we dont access frequently..
3 month old logs -> Glacier instant. not frquent access, more cheaper
6 months old logs -> glacier flexible Retrieval -> retrievals are in minutes
1 year old logs -> Glacier Deep archieve -> storage is more and more cheaper, but retrievals are more costly..

Glacier -> Retrieval + Restore

Intelligent-Tiering -> AWS monitors our objects and their access frquency. then AWS decides which tier is suitable. AWS charges for monitoring objects..

if you are not accessing frequently -> glacier
if you are accessing frequently -> standard

versions
========
if present config or state file corrupted, we can get previous versions

Delete Marker
=============
if we delete object in S3, AWS dont delete it really, it will just place delete marker on the object. in UI it looks like deleted, but actually it is not deleted.

It is safety feature for accidental deletes.

If you really want to delete the file? delete versions.. irreversable

100 devops engineers + 10 apply -> 1000 versions
300 days -> 3,00,000 versions -> 3,00,000 files

Lifecycle rules
==============
we can configure how many versions are required, very old versions can be automatically deleted by AWS

1 day -> 1000 versions
AWS will not delete immidiate. should be 1 day old. 

expire non current versions after certain days and certain number of old versions

30 days old -> standard-IA -> 60 days -> move to glacier instant -> 90 days -> glacier flexible retrieval -> 120 days -> glacier deep

AWS S3 stores data regionally, for DR keep the replication cross region replication

Cross Region Replication
=============
1. Create bucket in another region
2. Versioning should be on in both source and destination buckets

Upcoming files only will be replicated, not existing files. 

replicas will not be done immidiately.

Same region Replication
==============
if one account is compromised, attacker accessed S3 bucket and deleted... SRR bucket can save us..
