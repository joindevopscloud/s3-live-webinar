MFA Delete
==========
very very important files, you dont want anyone to delete those files even admin access

Who can enable/disable MFA Delete ? -> only root user
Only root user have access to delete objects
MFA enable is not available through UI, it can be only enabled through SDK or CLI


arn:aws:iam::160885265516:mfa/s25

aws s3api put-bucket-versioning \
  --bucket joindevops-160885265516-us-east-1-an \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn:aws:iam::160885265516:mfa/s25 681977" \
  --profile root-account
  
aws configure --profile root-account

Mfa Authentication is not supported on a bucket with lifecycle configuration.
lifecycle rules -> to expire un-necessary objects
MFA Authentication and lifecycle rules can't work together, you need to choose either one

How to keep terraform state secure?
===================================
1. We will have one EC2 instance just to terraform, install terraform
2. That instance will have admin access. IAM Role
3. IAM devops team group will have only read access to the files
4. Versionining on
5. Encryption -> at rest and at transit
6. Block public access -> on
7. Delete only root user with MFA -> bucket policy
8. don't enable MFA delete, because lifecycle rules will not work
9. if lifecycle rules will not work, thousands of state files and lock files with older versions will exist. so configure lifecycle rules
